# Políticas importantes – garantías, cancelaciones y confidencialidad

DTE trabaja con políticas simples para proteger al cliente y al equipo, y para que el proyecto avance sin ambigüedades. En términos de garantías, DTE garantiza calidad de implementación y cumplimiento de entregables acordados. No se garantiza un resultado específico (por ejemplo, “X ventas”), porque depende de variables externas: oferta, producto, estacionalidad, capacidad de atención y mercado. Lo que sí se garantiza es el proceso profesional: estrategia, ejecución, medición y optimización.

Sobre cancelaciones: en servicios mensuales, el cliente puede cancelar con aviso previo de [X días / completar]. Si el pago ya se realizó, se cubre el período activo y luego se detiene el trabajo. En proyectos, si el cliente cancela a mitad de camino, se entrega lo avanzado hasta el punto correspondiente al pago realizado, y se evalúan costos por trabajo ya ejecutado. Esto evita situaciones injustas para ambas partes.

En confidencialidad, DTE trata toda información del cliente como privada: accesos, métricas, bases de datos, estrategias, creatividades internas y procesos. No se comparte información con terceros sin autorización. Si el cliente requiere un acuerdo formal de confidencialidad (NDA), se puede firmar: [completar modalidad]. En proyectos con data sensible, DTE recomienda definir roles y permisos, y usar gestores de contraseñas o accesos temporales cuando sea posible.

Sobre propiedad de activos: por regla, las cuentas (Meta Ads, dominios, hosting) deben estar a nombre del cliente, y DTE opera con permisos. El código o entregables se definen según contrato: en la mayoría de casos, lo desarrollado para el cliente queda para el cliente, excepto herramientas propietarias o plantillas internas. Esto se aclara por escrito.

Estas políticas permiten que el agente responda preguntas difíciles con calma: “sí, podemos pausar”, “sí, cuidamos confidencialidad”, “sí, trabajamos con contrato”, y siempre con transparencia.
